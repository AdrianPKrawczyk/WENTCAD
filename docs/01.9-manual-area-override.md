# Specyfikacja: Ręczne Nadpisywanie Powierzchni (Manual Area Override)

## 1. Kontekst i Problem
W aplikacjach inżynierskich HVAC (Krok 1.5 - Bilans Powietrza oraz Krok 2.4/2.5 - Rysowanie Stref), powierzchnia wyliczona bezpośrednio z poligonu architektonicznego (np. obrys ścian) nie zawsze odpowiada rzeczywistej powierzchni bilansowej. Często obrysy CAD ignorują słupy konstrukcyjne, szachty instalacyjne czy wnęki. 

Dlatego konieczne jest umożliwienie inżynierowi **"odpięcia"** powierzchni strefy od surowej geometrii CAD i wprowadzenia własnej wartości ręcznie, bez utraty informacji o pierwotnym obrysie.

## 2. Architektura Rozwiązania (Wzorzec: Single Source of Truth)
Zamiast tworzyć osobne zmienne w silniku fizycznym, zachowujemy zmienną `area` jako jedyne źródło prawdy dla obliczeń (`PhysicsEngine`). Wprowadzamy natomiast mechanizm flagowania i przechowywania wartości z CAD w tle.

### 2.1 Zmiany w Modelu Danych (`src/types.ts`)
Interfejs `ZoneData` zostaje rozbudowany o:
* `area: number` - **OSTATECZNA** powierzchnia używana do obliczeń (bez zmian w logice silnika).
* `geometryArea: number | null` - Powierzchnia wyliczona czysto z poligonu CAD na Canvasie (tylko do odczytu / podglądu).
* `isAreaManual: boolean` - Flaga. `True` oznacza, że użytkownik wpisał powierzchnię ręcznie. Jeśli `True`, zmiany poligonu na Canvasie nie aktualizują zmiennej `area`. Zastępuje starą flagę `isAreaLinkedToGeometry`.

### 2.2 Zmiany w Interfejsie Użytkownika (UI)
1. **Panel Właściwości (`ZonePropertiesPanel.tsx`)**:
   * Zmiana inputu "Powierzchnia" - dodanie checkboxa `[MANUAL]` (analogicznie do `isTargetACHManual` czy `isMaxDbAManual`).
   * Zmiana koloru tła inputu na żółtawy (ostrzegawczy), gdy tryb MANUAL jest aktywny.
   * Dodanie małego tekstu pomocniczego poniżej inputu: *"Z obrysu CAD: X m²"*, jeśli `geometryArea` nie jest `null`.

2. **Tabela Bilansu (`AirBalanceTable.tsx`)**:
   * Kolumna `area`: 
     * Tło jasnoniebieskie (`#e0f2fe`) - gdy powierzchnia pochodzi z CAD (`geometryArea !== null` i `!isAreaManual`).
     * Tło jasnożółte (`#fefcbf`) - gdy powierzchnia jest wpisana ręcznie (`isAreaManual === true`).
     * Edytowalność włączona TYLKO gdy `isAreaManual === true` lub gdy w ogóle nie ma przypisanej geometrii.
   * Nowa kolumna `isAreaManual` (Checkbox) - umożliwiająca masowe włączanie/wyłączanie nadpisywania.

### 2.3 Reguły Logiczne dla Agenta (Integracja z Canvas)
Gdy Silnik Graficzny (Canvas) generuje lub modyfikuje obrys strefy, wysyła nową powierzchnię (np. `newCalculatedArea`). Store musi zachować się następująco:
```typescript
const updates: Partial<ZoneData> = { geometryArea: newCalculatedArea };
if (!zone.isAreaManual) {
   updates.area = newCalculatedArea; // Aktualizuj główną zmienną tylko, gdy nie ma blokady ręcznej
}
updateZone(zone.id, updates);
