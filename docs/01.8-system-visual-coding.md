# 01.8.1-system-visual-coding-manual.md - Zaawansowane Style Systemów

## 1. CEL
Wizualna identyfikacja systemów w tabeli i na rysunku z pełną kontrolą użytkownika nad priorytetami wyświetlania.

## 2. ROZSZERZONY MODEL STYLU (SystemStyle)
Każdy unikalny `systemId` posiada obiekt stylu:
- `color`: string | null (HEX)
- `pattern`: 'NONE' | 'DOTS' | 'HATCH'
- `isColorPriority`: boolean (Czy ten system ma narzucać swój kolor w pomieszczeniu?)
- `isPatternPriority`: boolean (Czy ten system ma narzucać swój wzór?)

## 3. LOGIKA ROZSTRZYGANIA KONFLIKTÓW (Priority & Fallback)
Jeśli pomieszczenie posiada dwa systemy (Nawiew S i Wywiew E), styl wiersza/strefy wyliczany jest następująco:

### A. Wybór Koloru:
1. Jeśli S ma `isColorPriority` i posiada `color` -> wybierz kolor S.
2. Jeśli E ma `isColorPriority` i posiada `color` -> wybierz kolor E.
3. Jeśli tylko jeden z nich posiada przypisany kolor -> wybierz ten kolor (Fallback).
4. Jeśli oba posiadają kolor i żaden nie ma priorytetu -> wybierz kolor Nawiewu (S) jako domyślny.

### B. Wybór Desenia (Pattern):
1. Analogiczna logika do koloru, używająca flagi `isPatternPriority` i pola `pattern`.

## 4. FUNKCJE DODATKOWE
- **Global Toggle:** Przełącznik "Włącz kolorowanie systemów" w ustawieniach widoku tabeli.
- **Auto-Generator:** Funkcja "Generuj kolory", która przypisuje unikalne barwy (bez wzorów) do wszystkich systemów, które nie mają jeszcze zdefiniowanego stylu.
- **Styling Presets:** Możliwość zapisu całej tablicy stylów (Mapy Systemów) pod nazwą (np. "Schemat Branżowy", "Schemat Inwestorski").
