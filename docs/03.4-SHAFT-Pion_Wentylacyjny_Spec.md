## 1. Definicja i typy

**SHAFT** (Pion wentylacyjny) to element łączący instalację wentylacyjną między kondygnacjami. Reprezentuje przewód pionowy przechodzący przez stropy budynku.

### Typy SHAFT

| Typ | Kierunek przepływu | Ikona | ComponentType |
|-----|-------------------|-------|---------------|
| `SHAFT_UP` | DO GÓRY | ↑ | Elementy poziome wychodzą do góry |
| `SHAFT_DOWN` | W DÓŁ | ↓ | Elementy poziome schodzą w dół |
| `SHAFT_THROUGH` | PRZELOTOWY | ↕ | Przepływ przechodzi przez pion |

### Kategoria komponentu

Wszystkie typy SHAFT mają `componentCategory = 'SHAFT'`.

## 3. Właściwości synchronizowane

**Zasada:** Zmiana tych właściwości na jednym węźle SHAFT automatycznie aktualizuje WSZYSTKIE węzły z tym samym `shaftId`.

### Lista właściwości synchronizowanych

| Właściwość | Typ | Opis | Przykład |
|------------|-----|------|----------|
| `shaftId` | `string` | Identyfikator pionu | "P1", "P2", "P3" |
| `shaftAutoNumber` | `number` | Numer automatyczny | 1, 2, 3 |
| `shaftRange` | `{fromFloorId: string, toFloorId: string}` | Zakres kondygnacji | {from: "floor-0", to: "floor-3"} |
| `systemId` | `string` | System HVAC | "NW1" (Nawiew) |
| `ahuId` | `string` | ID centrali | "" |
| `componentType` | `ComponentType` | Typ SHAFT | "SHAFT_UP" |
| `shaftShiftX` | `number` | Przesunięcie X | 0 (piksele) |
| `shaftShiftY` | `number` | Przesunięcie Y | 0 (piksele) |

### Implementacja synchronizacji

**Funkcja `syncShaftProperties` (useDuctStore.ts ~linia 730):**
```typescript
syncShaftProperties: (sourceNodeId, updates) => {
  const state = get();
  const sourceNode = state.nodes[sourceNodeId];
  if (!sourceNode || sourceNode.componentCategory !== 'SHAFT' || !sourceNode.shaftId) return;

  const newNodes = { ...state.nodes };
  const shaftId = updates.shaftId ?? sourceNode.shaftId;

  // Znajdź wszystkie węzły SHAFT z tym samym shaftId i zaktualizuj
  Object.values(state.nodes).forEach(node => {
    if (node.componentCategory === 'SHAFT' && node.shaftId === shaftId && node.id !== sourceNodeId) {
      newNodes[node.id] = {
        ...newNodes[node.id],
        ...updates,  // Rozpocznij wszystkie właściwości z updates
      };
    }
  });

  set({ nodes: newNodes });
}
```

### Przykład synchronizacji

```
Przed zmianą shaftId:
  Kondygnacja 0: SHAFT P1
  Kondygnacja 1: SHAFT P1
  Kondygnacja 2: SHAFT P1

Użytkownik zmienia nazwę na Kondygnacji 0 na "P2":

Po zmianie:
  Kondygnacja 0: SHAFT P2 ← źródło zmiany
  Kondygnacja 1: SHAFT P2 ← zsynchronizowane
  Kondygnacja 2: SHAFT P2 ← zsynchronizowane
```

## 5. Synchronizacja pozycji

### Zasada działania

Pozycja węzłów SHAFT może być **synchronizowana** lub **zablokowana**.

```
Synchronizacja WŁĄCZONA:
  Wszystkie węzły P1 mają tę samą pozycję (x, y)
  
Synchronizacja WYŁĄCZONA (isPositionManuallySet = true):
  Wybrane węzły mają własną pozycję, niezależną od innych
```

### Flaga `isPositionManuallySet`

**Typ:** `boolean` w `DuctNode`

**Domyślna wartość:** `false` (synchronizacja włączona)

**Kiedy ustawiana na `true`:**
- Gdy użytkownik przeciągnie węzeł SHAFT myszą
- Workspace2D.tsx ~linia 2034-2044

```typescript
onDragEnd={(e) => {
  if (currentTool === null) {
    const newX = e.target.x();
    const newY = e.target.y();
    // ...
    if (node.componentCategory === 'SHAFT') {
      updateDuctNode(node.id, { 
        x: newX, 
        y: newY, 
        isPositionManuallySet: true  // ← FLAGA USTAWIONA
      });
    }
  }
}}
```

### Scenariusze synchronizacji

#### Scenariusz 1: Synchronizacja WŁĄCZONA (domyślna)

```
Wynik tworzenia P1 z zakresem Parter → +3 Piętro:

Kondygnacja 0 (Parter):    SHAFT P1 na (x=100, y=200) ← węzeł źródłowy
Kondygnacja 1 (+1 Piętro):  SHAFT P1 na (x=100, y=200) ← synchronizowana
Kondygnacja 2 (+2 Piętro):  SHAFT P1 na (x=100, y=200) ← synchronizowana
Kondygnacja 3 (+3 Piętro):  SHAFT P1 na (x=100, y=200) ← synchronizowana
```

#### Scenariusz 2: Synchronizacja CZĘŚCIOWO WYŁĄCZONA

```
Kondygnacja 0 (Parter):    SHAFT P1 na (x=100, y=200) ← węzeł źródłowy
Kondygnacja 1 (+1 Piętro): SHAFT P1 na (x=150, y=250) ← ZABLOKOWANA (isPositionManuallySet=true)
Kondygnacja 2 (+2 Piętro): SHAFT P1 na (x=100, y=200) ← synchronizowana
Kondygnacja 3 (+3 Piętro): SHAFT P1 na (x=100, y=200) ← synchronizowana
```

#### Scenariusz 3: Włączenie synchronizacji

```
Użytkownik klika "Włącz synchronizację pozycji" w panelu właściwości

Rezultat:
  isPositionManuallySet = false na WSZYSTKICH węzłach P1
  Pozycja wszystkich węzłów = pozycja węzła źródłowego (x=100, y=200)

Kondygnacja 0 (Parter):    SHAFT P1 na (x=100, y=200)
Kondygnacja 1 (+1 Piętro): SHAFT P1 na (x=100, y=200) ← odblokowana
Kondygnacja 2 (+2 Piętro): SHAFT P1 na (x=100, y=200)
Kondygnacja 3 (+3 Piętro): SHAFT P1 na (x=100, y=200)
```

### Implementacja

**Funkcja `resetPositionSync` (useDuctStore.ts ~linia 746):**
```typescript
resetPositionSync: (sourceNodeId) => {
  const state = get();
  const sourceNode = state.nodes[sourceNodeId];
  if (!sourceNode || sourceNode.componentCategory !== 'SHAFT' || !sourceNode.shaftId) return;

  const shaftId = sourceNode.shaftId;
  const shaftShiftX = sourceNode.shaftShiftX || 0;
  const shaftShiftY = sourceNode.shaftShiftY || 0;
  const newNodes = { ...state.nodes };

  Object.values(state.nodes).forEach(node => {
    if (node.componentCategory === 'SHAFT' && node.shaftId === shaftId) {
      if (node.id !== sourceNodeId && !node.isPositionManuallySet) {
        // Aktualizuj pozycję jeśli nie była ręcznie ustawiona
        newNodes[node.id] = {
          ...newNodes[node.id],
          isPositionManuallySet: false,
          x: sourceNode.x + shaftShiftX,
          y: sourceNode.y + shaftShiftY,
        };
      } else {
        // Tylko resetuj flagę
        newNodes[node.id] = {
          ...newNodes[node.id],
          isPositionManuallySet: false,
        };
      }
    }
  });

  set({ nodes: newNodes });
}
```

## 6. Krawędzie pionowe

### Definicja

Krawędź pionowa to połączenie między węzłami SHAFT na sąsiadujących kondygnacjach.

### Automatyczne tworzenie

System automatycznie tworzy krawędzie pionowe gdy:
1. Tworzony jest nowy węzeł SHAFT na kondygnacji w zakresie
2. Istnieje węzeł SHAFT na sąsiedniej kondygnacji

### Struktura wizualna

```
                    ┌─ [AHU na dachu]
                    │
              [P1]──┤  ← krawędź pionowa (+2 Piętro → +3 Piętro)
                    │
[ANEMOSTAT]──[P1]──┤  ← krawędź pionowa (+1 Piętro → +2 Piętro)
              [P1]──┤  ← krawędź pionowa (Parter → +1 Piętro)
                    │
[ANEMOSTAT]──[P1]──┘
```

### Właściwości krawędzi pionowej

| Właściwość | Typ | Opis | Uwagi |
|------------|-----|------|-------|
| `id` | `string` | Unikalny ID | Generowany jako `edge-${crypto.randomUUID()}` |
| `sourceNodeId` | `string` | Węzeł na niższej kondygnacji | |
| `targetNodeId` | `string` | Węzeł na wyższej kondygnacji | |
| `systemId` | `string` | Dziedziczony z SHAFT | |
| `ahuId` | `string` | Dziedziczony z SHAFT | |
| `length` | `number` | Długość [m] | Do obliczenia z elevation kondygnacji |
| `shape` | `DuctShape` | Kształt | `'CIRCULAR'` lub `'RECTANGULAR'` |
| `roughness` | `number` | Chropowatość [m] | Domyślnie `0.00015` |

### Implementacja

**Funkcja `createVerticalEdge` (useDuctStore.ts ~linia 615):**
```typescript
createVerticalEdge: (sourceNodeId, targetNodeId, systemId, ahuId) => {
  set((state) => {
    // Sprawdź czy krawędź już istnieje
    const existingEdge = Object.values(state.edges).find(
      e => (e.sourceNodeId === sourceNodeId && e.targetNodeId === targetNodeId) ||
           (e.sourceNodeId === targetNodeId && e.targetNodeId === sourceNodeId)
    );
    
    if (existingEdge) return state; // Krawędź już istnieje
    
    const verticalEdgeId = `edge-${crypto.randomUUID()}`;
    return {
      edges: {
        ...state.edges,
        [verticalEdgeId]: {
          id: verticalEdgeId,
          sourceNodeId,
          targetNodeId,
          systemId,
          ahuId,
          length: 0, // Do obliczenia z elevation
          shape: 'CIRCULAR',
          roughness: 0.00015,
          internalInsulationThickness: 0,
          externalInsulationThickness: 0,
          velocity: 0,
          pressureDropLin: 0,
        },
      },
    };
  });
}
```

## 7. Osierocone węzły SHAFT (Orphaned Nodes)

### Definicja

Osierocony węzeł to węzeł SHAFT z danym `shaftId`, który znajduje się na kondygnacji **poza** aktualnym zakresem `shaftRange`.

### Kiedy powstają

Gdy użytkownik zmieni zakres pionu:

```
Było: P1 na Kondygnacji 0, 1, 2, 3
Zmieniono zakres: Parter → +1 Piętro

Wynik: 
  - Kondygnacja 0, 1: Węzły w zakresie ✓
  - Kondygnacja 2, 3: OSIEROCONE węzły P1 ✗
```

### Funkcja wykrywania osieroconych

**`getOrphanedShaftNodes` (useDuctStore.ts ~linia 567):**
```typescript
getOrphanedShaftNodes: (shaftId, shaftRange) => {
  const state = get();
  const allShaftsWithId = Object.values(state.nodes).filter(
    node => node.componentCategory === 'SHAFT' && node.shaftId === shaftId
  );

  if (!shaftRange || !shaftRange.fromFloorId || !shaftRange.toFloorId) {
    return allShaftsWithId; // Brak zakresu = wszystkie nie są osierocone
  }

  return allShaftsWithId.filter(node => {
    // Węzeł jest osierocony jeśli NIE jest na fromFloorId ANI na toFloorId
    return node.floorId !== shaftRange.fromFloorId && node.floorId !== shaftRange.toFloorId;
  });
}
```

### Opcje zarządzania osieroconymi

System wyświetla modal **OrphanedShaftModal** z opcjami:

| Opcja | Opis | Efekt |
|-------|------|-------|
| **USUŃ** | Trwale usuwa osierocone węzły i krawędzie | `removeOrphanedShaftNodes()` |
| **POZOSTAW** | Zostawia osierocone elementy | Brak akcji - zostają jako osobne węzły |
| **UTWÓRZ P2** | Przypisuje nowy identyfikator | `reassignShaftNodes()` z nowym `shaftId` |
| **PRZENIEŚ DO P1** | Dołącza do istniejącego pionu | `reassignShaftNodes()` z istniejącym `shaftId` |

### Implementacja modala

**OrphanedShaftModal.tsx** - komponent React z:
- Listą osieroconych kondygnacji
- Radio buttons dla opcji
- Przyciskiem "Zastosuj"
- Walidacją (np. "PRZENIEŚ" wymaga wyboru celu)

## 8. Widoczność na kondygnacjach

### Zasada

**SHAFT jest widoczny na WSZYSTKICH kondygnacjach w zakresie `shaftRange`.**

```
Zakres: Parter → +3 Piętro (4 kondygnacje)

Widoczność:
┌─────────────┐
│ +3 Piętro   │  ← P1 WIDOCZNY (węzeł istnieje)
├─────────────┤
│ +2 Piętro   │  ← P1 WIDOCZNY (węzeł istnieje)
├─────────────┤
│ +1 Piętro   │  ← P1 WIDOCZNY (węzeł istnieje)
├─────────────┤
│ Parter      │  ← P1 WIDOCZNY (węzeł źródłowy)
└─────────────┘
```

### Tworzenie węzłów na wszystkich kondygnacjach

**`syncShaftToAllFloorsInRange` (DuctPropertiesPanel.tsx ~linia 46-105):**
```typescript
function syncShaftToAllFloorsInRange(
  sourceNodeId,
  allFloors,
  createShaftNodeOnFloor,
  createVerticalEdge,
  nodes
) {
  const sourceNode = nodes[sourceNodeId];
  const shaftRange = sourceNode.shaftRange;
  
  // Pobierz wszystkie kondygnacje posortowane po elevation
  const sortedFloors = Object.values(allFloors).sort((a, b) => a.elevation - b.elevation);
  
  // Filtruj kondygnacje w zakresie [min, max]
  const minElevation = Math.min(fromFloor.elevation, toFloor.elevation);
  const maxElevation = Math.max(fromFloor.elevation, toFloor.elevation);
  
  const floorsInRange = sortedFloors.filter(
    f => f.elevation >= minElevation && f.elevation <= maxElevation
  );
  
  // Utwórz węzły SHAFT na każdej kondygnacji w zakresie
  for (const floor of floorsInRange) {
    if (floor.id !== sourceNode.floorId) {
      createShaftNodeOnFloor(sourceNode, floor.id, sourceNode.shaftId);
    }
  }
  
  // Utwórz krawędzie pionowe między sąsiednimi kondygnacjami
  for (let i = 0; i < floorsInRange.length - 1; i++) {
    // ... logika tworzenia krawędzi
  }
}
```
