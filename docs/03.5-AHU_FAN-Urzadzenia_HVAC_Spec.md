## 1. Definicja i typy

**AHU** (Air Handling Unit - Centrala wentylacyjna) i **FAN** (Wentylator) to elementy wyposażenia systemu wentylacyjnego. Reprezentują fizyczne urządzenia na planie budynku.

### Typy urządzeń

| Typ | Opis | Kategoria | Ikona |
|-----|------|-----------|-------|
| `AHU` | Centrala wentylacyjna | EQUIPMENT | Prostokąt z "AHU" |
| `FAN` | Wentylator | EQUIPMENT | Prostokąt z "FAN" |
| `HEAT_RECOVERY` | Rekuperator (z odzyskiem ciepła) | EQUIPMENT | Prostokąt z "HR" |

### Kategoria komponentu

Wszystkie typy urządzeń mają `componentCategory = 'EQUIPMENT'`.

### Charakterystyka

- **AHU/FAN są pojedynczymi węzłami** - nie są synchronizowane między kondygnacjami
- **Mają wymiary fizyczne** - użytkownik definiuje rzeczywiste wymiary w centymetrach
- **Mogą być podłączone do przewodów** - posiadają porty na wszystkich 4 stronach
- **Zawierają parametry wydajności** - przepływ nominalny, ciśnienie, moc

## 2. Tworzenie AHU/FAN

### Przepływ tworzenia

```
KROK 1: Użytkownik wybiera narzędzie AHU lub FAN z toolbar'a HVAC
         → Workspace2D.tsx: activeDuctTool = 'AHU' | 'FAN' | 'HEAT_RECOVERY'

KROK 2: Użytkownik klika na canvasie
         → Workspace2D.tsx: onClick handler tworzy węzeł w miejscu kliknięcia
         → useDuctStore.ts: addDuctNode() z componentCategory='EQUIPMENT'

KROK 3: System tworzy węzeł z domyślnymi wymiarami
         → Domyślne wymiary:
           - widthCm: 50 (cm)
           - heightCm: 30 (cm)
           - lengthCm: 100 (cm)
         → Domyślne parametry wydajności:
           - ratedFlow: 0 (m³/h)
           - ratedPressure: 0 (Pa)
           - powerConsumption: 0 (W)

KROK 4: Użytkownik otwiera panel właściwości (DuctPropertiesPanel.tsx) i:
         - Ustawia wymiary fizyczne (szerokość, wysokość, długość)
         - Ustawia parametry wydajności (przepływ, ciśnienie)
         - Podłącza przewody do portów
```

### Kod źródłowy

**Tworzenie węzła (Workspace2D.tsx ~linia 1296-1332):**
```typescript
if (activeDuctTool && currentTool === null) {
  const canvasPos = {
    x: (pointerPos.x - position.x) / scale,
    y: (pointerPos.y - position.y) / scale,
  };
  
  addDuctNode({
    id: nodeId,
    type: 'BRANCH',
    componentCategory: getCategoryForDuctTool(activeDuctTool), // 'EQUIPMENT'

    systemId,
    ahuId: '',
    x: canvasPos.x,
    y: canvasPos.y,
    floorId: activeFloorId,
    flow: 0,
    pressureDropLocal: 0,
    soundPowerLevel: [0, 0, 0, 0, 0, 0, 0, 0],
    // Domyślne wymiary:
    widthCm: 50,
    heightCm: 30,
    lengthCm: 100,
    ratedFlow: 0,
    ratedPressure: 0,
  });
}
```

## 3. Wymiary fizyczne

### Definicja wymiarów

AHU/FAN mają **wymiary fizyczne** przechowywane w centymetrach:

| Właściwość | Typ | Opis | Jednostka | Domyślna wartość |
|------------|-----|------|-----------|------------------|
| `widthCm` | `number` | Szerokość | cm | 50 |
| `heightCm` | `number` | Wysokość | cm | 30 |
| `lengthCm` | `number` | Długość (głębokość) | cm | 100 |

### Przykładowe wymiary

| Typ urządzenia | Szerokość (cm) | Wysokość (cm) | Długość (cm) |
|----------------|----------------|---------------|--------------|
| AHU mały | 100 | 60 | 150 |
| AHU średni | 200 | 80 | 250 |
| AHU duży | 300 | 120 | 400 |
| FAN kanałowy | 40 | 40 | 60 |
| FAN promieniowy | 60 | 60 | 80 |

### Wymiary na planie 2D

Na planie 2D renderowane są tylko **szerokość** i **wysokość**:
- `widthCm` → poziomy wymiar prostokąta
- `heightCm` → pionowy wymiar prostokąta

`lengthCm` (długość) nie jest renderowana na planie 2D - reprezentuje głębokość urządzenia widoczną na przekroju.

## 4. Parametry wydajności

### Lista parametrów

| Właściwość | Typ | Opis | Jednostka | Domyślna wartość |
|------------|-----|------|-----------|------------------|
| `ratedFlow` | `number` | Nominalny przepływ powietrza | m³/h | 0 |
| `ratedPressure` | `number` | Nominalne ciśnienie dyspozycyjne | Pa | 0 |
| `powerConsumption` | `number` | Pobór mocy elektrycznej | W | 0 |
| `efficiency` | `number` | Sprawność energetyczna | % | 0 |
| `soundPowerLevel` | `number[]` | Poziom mocy akustycznej | dB | [0,0,0,0,0,0,0,0] |

### Opis parametrów

**ratedFlow (Przepływ nominalny)**
- Maksymalny przepływ powietrza przez urządzenie
- Jednostka: m³/h (metr sześcienny na godzinę)
- Typowe wartości: 500 - 50000 m³/h

**ratedPressure (Ciśnienie nominalne)**
- Ciśnienie dyspozycyjne wentylatora
- Jednostka: Pa (Paskale)
- Typowe wartości: 100 - 1000 Pa

**powerConsumption (Pobór mocy)**
- Moc elektryczna pobierana przez urządzenie
- Jednostka: W (Waty)
- Typowe wartości: 500 - 15000 W

**efficiency (Sprawność)**
- Dla rekuperatorów: skuteczność odzysku ciepła
- Jednostka: % (procent)
- Typowe wartości: 60 - 95%

**soundPowerLevel (Poziom mocy akustycznej)**
- Poziom mocy akustycznej w pasmach oktawowych
- Tablica 8 wartości dla częstotliwości: 63, 125, 250, 500, 1000, 2000, 4000, 8000 Hz
- Jednostka: dB

## 5. Renderowanie na canvasie

### Kształt na canvasie

AHU/FAN renderowane są jako **zaokrąglony prostokąt** z etykietą:

```
┌────────────────────┐
│                    │
│        AHU         │  ← Etykieta typu urządzenia
│                    │
└────────────────────┘
```

### Właściwości wizualne

| Element | Styl | Uwagi |
|---------|------|-------|
| Kształt | Prostokąt z zaokrąglonymi rogami | cornerRadius: 4px |
| Kolor fill | `nodeColor` lub biały gdy aktywny/wybrany | Zależy od systemu |
| Kolor stroke | `nodeColor` | Zależy od systemu |
| Grubość stroke | `strokeWidth` | 2px |
| Etykieta | "AHU", "FAN", "HR" | Tekst bold, wycentrowany |

### Implementacja renderowania

**Workspace2D.tsx ~linia 1713-1740:**
```typescript
// EQUIPMENT - RoundedRect (AHU, FAN) - SKALOWANE wymiary fizyczne
if (node.componentCategory === 'EQUIPMENT') {
  const floorScale = scaleFactor || 1; // px/cm
  const eqWidth = (node.widthCm || 50) * floorScale;
  const eqHeight = (node.heightCm || 30) * floorScale;
  const isAHU = node.componentType === 'AHU';
  
  return (
    <Group>
      <Rect
        x={-eqWidth / 2}
        y={-eqHeight / 2}
        width={eqWidth}
        height={eqHeight}
        cornerRadius={4}
        fill={isActive || isSelected ? '#ffffff' : nodeColor}
        stroke={nodeColor}
        strokeWidth={strokeWidth}
      />
      <Text
        text={isAHU ? 'AHU' : 'FAN'}
        fontSize={fontSize}
        fill={isActive || isSelected ? nodeColor : '#ffffff'}
        fontStyle="bold"
        align="center"
        verticalAlign="middle"
      />
    </Group>
  );
}
```

## 6. Formuła skalowania

### Zasada

AHU/FAN na planie 2D renderowane są w **rzeczywistych wymiarach fizycznych** przeliczonych na piksele.

### Formuła

```
canvasWidth = widthCm × scaleFactor    [px]
canvasHeight = heightCm × scaleFactor [px]

 gdzie:
   - scaleFactor = piksele na centymetr (px/cm)
   - Zależy od skali rysunku (np. 1:50 → scaleFactor = 2)
```

### Przykład obliczeń

```
Dane:
  widthCm = 200 cm
  heightCm = 80 cm
  scaleFactor = 2 px/cm (odpowiada skali 1:50)

Wynik:
  canvasWidth = 200 × 2 = 400 px
  canvasHeight = 80 × 2 = 160 px
```

### Niezależność od zoomu

**UWAGA:** Wymiary AHU/FAN **NIE SĄ** mnożone przez `stage.scale()`.

- `stage.scale()` kontroluje zoom canvasu (widok)
- `scaleFactor` kontroluje rzeczywiste wymiary na planie

```
Canvas (zoom 100%):         Canvas (zoom 200%):
┌──────────────┐            ┌──────────────┐
│   AHU 400px │            │   AHU 400px  │  ← Ten sam rozmiar!
│   x 160px   │            │   x 160px    │
└──────────────┘            └──────────────┘
      ↑                            ↑
  scaleFactor ×2              scaleFactor ×2
```

## 7. Porty połączeniowe

### Definicja portów

AHU/FAN posiadają **4 porty połączeniowe** - po jednym na każdej stronie:

| Port | Kierunek | Opis |
|------|----------|------|
| TOP | Góra | Połączenie górne |
| RIGHT | Prawo | Połączenie prawe |
| BOTTOM | Dół | Połączenie dolne |
| LEFT | Lewo | Połączenie lewe |

### Wizualizacja portów

```
              [PORT TOP]
                  │
                  ▼
    ┌─────────────────────────┐
    │                         │
[LEFT]        AHU         [RIGHT]
    │                         │
    │                         │
    └─────────────────────────┘
                  ▲
                  │
            [PORT BOTTOM]
```

### Implementacja portów

Porty są automatycznie wykrywane na podstawie najbliższej krawędzi:

**Workspace2D.tsx ~linia 1470-1550:**
```typescript
// Automatyczne wykrywanie portu na podstawie pozycji
const getConnectionPort = (node: DuctNode, clickX: number, clickY: number): string => {
  const nodeWidth = (node.widthCm || 50) * scaleFactor;
  const nodeHeight = (node.heightCm || 30) * scaleFactor;
  
  const dx = Math.abs(clickX - node.x);
  const dy = Math.abs(clickY - node.y);
  
  // Określenie najbliższego portu
  if (dy > dx) {
    // Pionowe połączenie (TOP lub BOTTOM)
    return clickY < node.y ? 'TOP' : 'BOTTOM';
  } else {
    // Poziome połączenie (LEFT lub RIGHT)
    return clickX < node.x ? 'LEFT' : 'RIGHT';
  }
};
```

### Drag & Drop

Przeciąganie AHU/FAN działa jak dla innych węzłów:

```typescript
onDragEnd={(e) => {
  if (currentTool === null) {
    const newX = e.target.x();
    const newY = e.target.y();
    updateDuctNode(node.id, { x: newX, y: newY });
  }
}}
```

### Zmiana wymiarów

Zmiana wymiarów w panelu właściwości wymaga ponownego renderowania:

```typescript
// DuctPropertiesPanel.tsx
const handleDimensionChange = (dimension: 'widthCm' | 'heightCm' | 'lengthCm', value: number) => {
  updateDuctNode(selectedNodeId, { [dimension]: value });
};
```
