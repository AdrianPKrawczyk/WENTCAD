# 02.6-export-framing-spec.md - Specyfikacja Mechanizmu Kadrowania Eksportu

> **Cel dokumentu:** Opis mechanizmu rysowania i eksportu z użyciem zdefiniowanych kadrów (export frames) w module CAD platformy WENTCAD.

---

## 1. Architektura Kadrowania

### 1.1 Przechowywanie Danych

Kadry eksportu są przechowywane jako tablica `exportRegions` w interfejsie `Floor`:

```typescript
// src/types.ts - Floor interface
export interface Floor {
  id: string;
  name: string;
  // ...
  exportRegions?: {
    id: string;
    name: string;
    x: number;      // Współrzędna X lewego górnego rogu [px sceny]
    y: number;      // Współrzędna Y lewego górnego rogu [px sceny]
    width: number;  // Szerokość kadru [px sceny]
    height: number; // Wysokość kadru [px sceny]
  }[];
}
```

**Jednostki współrzędnych:** Wszystkie wartości `x, y, width, height` są wyrażone w **pikselach sceny Konva** (scene pixels), NIE w pikselach ekranu. Oznacza to, że współrzędne uwzględniają już transformację skali (`scale`) i przesunięcia (`position`) sceny.

### 1.2 Rysowanie Kadru (CROP Tool)

Użytkownik aktywuje narzędzie "Kadr" (ikona `CROP`) na pasku narzędzi Canvas:

1. **Punkt początkowy:** `onMouseDown` - zapisuje `canvasPos` jako punkt początkowy (lew górny róg)
2. **Podgląd:** `onMouseMove` - rysuje dynamiczny prostokąt podczas przeciągania
3. **Zapis:** `onMouseUp` - po zwolnieniu przycisku:
   - Oblicza bezwzględne wartości `x, y, width, height`
   - Wyświetla `window.prompt()` z prośbą o nazwę kadru
   - Dodaje nowy obiekt do `exportRegions` w `Floor`

**Zapis do store:**
```typescript
const newRegion = {
  id: `crop-${Date.now()}`,
  name: regionName,
  x, y, width, height  // współrzędne sceny Konva
};
updateFloor(activeFloorId, {
  exportRegions: [...existingRegions, newRegion]
});
```

## 2. Mechanizm Eksportu PNG

### 2.1 Zasada Działania

Eksport PNG wykorzystuje metodę `stage.toDataURL()` z biblioteki Konva, która renderuje fragment sceny do formatu Base64 PNG.

**Kluczowa własność:** Współrzędne przekazane do `toDataURL({ x, y, width, height })` odnoszą się do **układu współrzędnych sceny Konva** (scene coordinates), a NIE do układu ekranu. Dzięki temu, niezależnie od poziomu zoomu czy przesunięcia widoku, można precyzyjnie wyeksportować dowolny fragment sceny.

### 2.2 Konfiguracja Eksportu

Współrzędne kadru z `exportRegions` MUSZĄ być przeliczone na układ sceny Konva:

```typescript
// Workspace2D.tsx - handleExportPNG
const handleExportPNG = useCallback(async (regionId: string, includeBackground: boolean) => {
  const region = activeFloorMetadata?.exportRegions?.find(r => r.id === regionId);
  if (!region || !stageRef.current) return;

  // Ukryj podkład jeśli użytkownik nie chce go w eksporcie
  if (!includeBackground) setForceHideUnderlay(true);
  
  // Odczekaj na re-render bez podkładu
  setTimeout(() => {
    const stage = stageRef.current!;
    
    // PRZELICZENIE WSPÓŁRZĘDNYCH:
    // region.x, region.y to współrzędne na "ekranie Canvas" (stage pixels)
    // Konieczne jest odjęcie pozycji sceny, aby uzyskać współrzędne sceny Konva
    const exportConfig = {
      x: region.x,                    // Współrzędna X względem lewej krawędzi Stage
      y: region.y,                    // Współrzędna Y względem górnej krawędzi Stage
      width: region.width,            // Szerokość w px
      height: region.height,          // Wysokość w px
      pixelRatio: 2                   // Współczynnik dla wysokiej rozdzielczości (retina)
    };

    const dataUrl = stage.toDataURL('image/png', 1.0, false, exportConfig);
    
    // Pobranie i zapis pliku
    const link = document.createElement('a');
    link.download = `${underlayName || 'WENTCAD'}_${region.name}_export.png`;
    link.href = dataUrl;
    document.body.appendChild(link);
    link.click();
    document.body.removeChild(link);
    
    setForceHideUnderlay(false);
  }, 100);
}, [...]);
```

### 2.3 Współczynnik Pikseli (pixelRatio)

Dla uzyskania wysokiej jakości obrazu (zalecane przy druku lub prezentacjach):
- `pixelRatio: 1` - rozdzielczość 1:1 (odpowiada rozmiarowi kadru)
- `pixelRatio: 2` - rozdzielczość podwójna (zalecane dla profesjonalnych wydruków)

### 2.4 Opcja "Dołącz Podkład"

Użytkownik może wyeksportować:
- **Z podkładem:** Scena renderowana normalnie z tłem PDF/IMG
- **Bez podkładu:** Przezroczyste tło (tylko strefy i metki)

Implementacja polega na ustawieniu flagi `forceHideUnderlay` przed eksportem, co powoduje, że warstwa podkładu (`KonvaImage`) ma `visible={false}`.

## 3. Mechanizm Eksportu DXF

### 3.1 Różnica w Porównaniu do PNG

W przeciwieństwie do PNG (który jest zrzutem rastrowym), eksport DXF wymaga:
1. **Filtrowania geometrii:** Tylko obiekty wewnątrz kadru powinny być eksportowane
2. **Przekształcenia współrzędnych:** Punkt (0,0) pliku DXF powinien odpowiadać lewemu górnemu rogowi kadru

### 3.2 Sygnatura Funkcji

```typescript
// src/lib/dxfExport.ts
export function exportToDXF(
  floorState: FloorCanvasState,
  zones: Record<string, ZoneData>,
  getTagText: (zoneId: string) => { col1: string; col2: string },
  exportFrame?: { x: number; y: number; width: number; height: number }
): string
```

### 3.3 Algorytm Filtrowania (AABB Intersection)

Dla każdego poligonu strefy sprawdzamy, czy przynajmniej jeden jego wierzchołek znajduje się wewnątrz zdefiniowanego prostokąta eksportu:

```typescript
function isPolygonInFrame(
  points: number[],  // tablica [x1, y1, x2, y2, ...]
  frame: { x: number; y: number; width: number; height: number }
): boolean {
  // Test kolizji AABB (Axis-Aligned Bounding Box)
  for (let i = 0; i < points.length; i += 2) {
    const px = points[i];
    const py = points[i + 1];
    
    if (
      px >= frame.x &&
      px <= frame.x + frame.width &&
      py >= frame.y &&
      py <= frame.y + frame.height
    ) {
      return true; // Przynajmniej jeden punkt jest wewnątrz kadru
    }
  }
  return false;
}
```

### 3.4 Transformacja Współrzędnych

Gdy kadrowanie jest aktywne, współrzędne DXF są przesuwane tak, aby lewy górny róg kadru stał się punktem (0,0) w pliku CAD:

```typescript
// src/lib/dxfExport.ts
export function exportToDXF(
  floorState: FloorCanvasState,
  zones: Record<string, ZoneData>,
  getTagText: (zoneId: string) => { col1: string; col2: string },
  exportFrame?: { x: number; y: number; width: number; height: number }
) {
  const dxf = new DxfWriter();
  
  // ... setup layers ...
  
  const origin = floorState.referenceOrigin || { x: 0, y: 0 };
  const scale = floorState.scaleFactor || 1; // meters per pixel
  
  // Offset kadrowania - punk (0,0) CAD odpowiada lewemu górnemu rogowi kadru
  const frameOffset = exportFrame 
    ? { x: exportFrame.x, y: exportFrame.y } 
    : { x: 0, y: 0 };
  
  // Transformacja: px sceny -> metry CAD z uwzględnieniem kadrowania
  const toCAD = (p: Point) => {
    return {
      x: (p.x - origin.x - frameOffset.x) * scale,
      y: -(p.y - origin.y - frameOffset.y) * scale // Y jest odwrócone w CAD
    };
  };
  
  // Filtrowanie polygonów
  const floorPolygons = floorState.polygons || [];
  
  floorPolygons.forEach(poly => {
    // Pomijamy poligony spoza kadru
    if (exportFrame && !isPolygonInFrame(poly.points, exportFrame)) {
      return;
    }
    
    // ... reszta logiki rysowania poligonu ...
  });
  
  return dxf.stringify();
}
```

### 3.5 Ograniczenia Cięcia Geometrii Wektorowej

**Ważne zastrzeżenia:**

1. **Test AABB jest przybliżony:** Poligon może mieć część wierzchołków wewnątrz kadru, a część na zewnątrz. W takim przypadku eksportujemy CAŁY poligon (nie obcinamy go geometrycznie). Użytkownik musi sam zdecydować, czy chce eksportować pełny obiekt.

2. **Metki poza kadrem:** Jeśli pozycja metki (`tagPosition` w `ZoneData`) znajduje się poza kadrem, metka NIE zostanie wyeksportowana (nawet jeśli poligon jest wewnątrz kadru).

3. **Podkład:** DXF NIE zawiera podkładu (PDF/IMG). Jest to celowe zachowanie - plik wektorowy zawiera tylko warstwy WENTCAD (obrysy, wypełnienia, metki).

## 4. Plik Wywoławczy (Workspace2D)

### 4.1 handleExportPNG

```typescript
const handleExportPNG = useCallback(async (regionId: string, includeBackground: boolean) => {
  const region = activeFloorMetadata?.exportRegions?.find(r => r.id === regionId);
  if (!region || !stageRef.current) return;

  if (!includeBackground) setForceHideUnderlay(true);

  setTimeout(() => {
    try {
      const stage = stageRef.current!;
      // Współrzędne kadru są już w układzie sceny Konva
      const dataUrl = stage.toDataURL({
        x: region.x,
        y: region.y,
        width: region.width,
        height: region.height,
        pixelRatio: 2
      });

      const link = document.createElement('a');
      link.download = `${underlayName || 'WENTCAD'}_${region.name}_export.png`;
      link.href = dataUrl;
      document.body.appendChild(link);
      link.click();
      document.body.removeChild(link);
      
      toast.success(`Wyeksportowano kadr: ${region.name}`);
    } catch (err) {
      console.error(err);
      toast.error('Błąd podczas generowania PNG.');
    } finally {
      setForceHideUnderlay(false);
      setIsExportModalOpen(false);
    }
  }, 100);
}, [...]);
```

## 6. Weryfikacja i Testy

### 6.1 Test PNG Export
1. Narysuj kadr na scenie z widocznymi strefami
2. Otwórz okno eksportu (przycisk w toolbarze tabeli)
3. Wybierz kadr i kliknij "Eksportuj do PNG"
4. Sprawdź, czy:
   - Obraz odpowiada dokładnie zaznaczonemu obszarowi
   - Strefy są wyraźne (opcja z/bez podkładu)
   - Rozdzielczość jest wystarczająca (pixelRatio: 2)

### 6.2 Test DXF Export
1. Zdefiniuj kadr obejmujący część stref
2. Wyeksportuj do DXF
3. Otwórz plik w AutoCAD/QCAD
4. Sprawdź, czy:
   - Tylko strefy wewnątrz kadru są widoczne
   - Współrzędne odpowiadają punktowi (0,0) w lewym górnym rogu kadru
   - Metki są poprawnie pozycjonowane
