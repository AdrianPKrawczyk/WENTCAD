# 02.6-export-framing-spec.md - Specyfikacja Mechanizmu Kadrowania Eksportu

> **Wersja:** 2.1 (Iteracja 2.10.2)
> **Data aktualizacji:** 2026-03-18
> **Cel dokumentu:** Opis mechanizmu rysowania i eksportu z użyciem zdefiniowanych kadrów (export frames) w module CAD platformy WENTCAD.

---

## 1. Architektura Kadrowania

### 1.1 Przechowywanie Danych

Kadry eksportu są przechowywane jako tablica `exportRegions` w interfejsie `Floor`:

```typescript
// src/types.ts - Floor interface
export interface Floor {
  id: string;
  name: string;
  // ...
  exportRegions?: {
    id: string;
    name: string;
    x: number;      // Współrzędna X lewego górnego rogu [px sceny]
    y: number;      // Współrzędna Y lewego górnego rogu [px sceny]
    width: number;  // Szerokość kadru [px sceny]
    height: number; // Wysokość kadru [px sceny]
  }[];
}
```

**Jednostki współrzędnych:** Wszystkie wartości `x, y, width, height` są wyrażone w **pikselach sceny Konva** (scene pixels), NIE w pikselach ekranu. Oznacza to, że współrzędne uwzględniają już transformację skali (`scale`) i przesunięcia (`position`) sceny.

### 1.2 Rysowanie Kadru (CROP Tool)

Użytkownik aktywuje narzędzie "Kadr" (ikona `CROP`) na pasku narzędzi Canvas:

1. **Punkt początkowy:** `onMouseDown` - zapisuje `canvasPos` jako punkt początkowy (lew górny róg)
2. **Podgląd:** `onMouseMove` - rysuje dynamiczny prostokąt podczas przeciągania
3. **Zapis:** `onMouseUp` - po zwolnieniu przycisku:
   - Oblicza bezwzględne wartości `x, y, width, height`
   - Wyświetla `window.prompt()` z prośbą o nazwę kadru
   - Dodaje nowy obiekt do `exportRegions` w `Floor`

**Zapis do store:**
```typescript
const newRegion = {
  id: `crop-${Date.now()}`,
  name: regionName,
  x, y, width, height  // współrzędne sceny Konva
};
updateFloor(activeFloorId, {
  exportRegions: [...existingRegions, newRegion]
});
```

## 2. Mechanizm Eksportu PNG

### 2.1 Zasada Działania

Eksport PNG wykorzystuje metodę `stage.toDataURL()` z biblioteki Konva, która renderuje fragment sceny do formatu Base64 PNG.

**Kluczowa własność:** Współrzędne przekazane do `toDataURL({ x, y, width, height })` odnoszą się do **układu współrzędnych sceny Konva** (scene coordinates), a NIE do układu ekranu. Dzięki temu, niezależnie od poziomu zoomu czy przesunięcia widoku, można precyzyjnie wyeksportować dowolny fragment sceny.

### 2.2 Algorytm Eksportu PNG (v2)

Ze względu na problemy z transformacją współrzędnych przy różnych poziomach zoomu, wdrożono następujący algorytm:

```typescript
// Workspace2D.tsx - handleExportPNG (v2)
const handleExportPNG = async (regionId: string, includeBackground: boolean) => {
  const region = activeFloorMetadata?.exportRegions?.find(r => r.id === regionId);
  if (!region || !stageRef.current) return;

  try {
    const stage = stageRef.current!;
    
    // 1. UKRYJ WARSTWĘ UI (siatka, ramki, znaczniki)
    if (uiLayerRef.current) {
      uiLayerRef.current.hide();
    }
    
    // 2. UKRYJ PODKŁAD JEŚLI NIE CHCIANY
    if (!includeBackground) {
      setForceHideUnderlay(true);
    }
    
    // 3. POCZEKAJ NA RE-RENDER I WYMUŚ REDRAW
    await new Promise(resolve => setTimeout(resolve, 50));
    stage.draw();
    
    // 4. ZAPISZ OBECNY WIDOK
    const oldScale = stage.scaleX();
    const oldPos = stage.position();
    
    // 5. PRZELICZ WSPÓŁRZĘDNE KADRU NA UKŁAD EKRANU
    // Przy scale=1, pozycja kadru w px ekranu będzie:
    const exportX = (region.x * oldScale) + oldPos.x;
    const exportY = (region.y * oldScale) + oldPos.y;
    const exportWidth = region.width * oldScale;
    const exportHeight = region.height * oldScale;
    
    // 6. RESETUJ WIDOK DO SKALI 1:1
    stage.scale({ x: 1, y: 1 });
    stage.position({ x: 0, y: 0 });
    stage.draw();
    
    // 7. EKSPORTUJ Z DOKŁADNYM KADREM
    const dataUrl = stage.toDataURL({
      x: exportX,
      y: exportY,
      width: exportWidth,
      height: exportHeight,
      pixelRatio: 3  // Wysoka jakość (3x rozdzielczość)
    });
    
    // 8. PRZYWRÓĆ WIDOK UŻYTKOWNIKA
    stage.scale({ x: oldScale, y: oldScale });
    stage.position(oldPos);
    
    // 9. POKAŻ WARSTWĘ UI
    if (uiLayerRef.current) {
      uiLayerRef.current.show();
    }
    if (!includeBackground) {
      setForceHideUnderlay(false);
    }
    
    stage.draw();
    
    // 10. Pobierz plik
    downloadFile(dataUrl, `${name}_export.png`);
  } catch (err) {
    // PRZYWRÓĆ STAN UI NAWET W PRZYPADKU BŁĘDU
    if (uiLayerRef.current) uiLayerRef.current.show();
    setForceHideUnderlay(false);
    if (stageRef.current) stageRef.current.draw();
  }
};
```

### 2.3 Struktura Warstw

Aby umożliwić czysty eksport bez elementów UI, sceny Konva są podzielone na warstwy:

| Warstwa | Ref | Zawartość |
|---------|-----|-----------|
| `ui` | `uiLayerRef` | Siatka (`drawGrid()`), ramki kadru, znaczniki punktu 0,0, linie kalibracji, pomiary |
| `content` | `contentLayerRef` | Strefy (poligony), obrysy CAD, metki, podkład PDF/IMG |

Przed eksportem warstwa `ui` jest ukrywana (`hide()`), a po eksporcie przywracana (`show()`).

### 2.4 Współczynnik Pikseli (pixelRatio)

Dla uzyskania wysokiej jakości obrazu (zalecane przy druku lub prezentacjach):
- `pixelRatio: 1` - rozdzielczość 1:1 (odpowiada rozmiarowi kadru)
- `pixelRatio: 2` - rozdzielczość podwójna
- `pixelRatio: 3` - rozdzielczość potrójna (zalecane dla profesjonalnych wydruków)

### 2.5 Opcja "Dołącz Podkład"

Użytkownik może wyeksportować:
- **Z podkładem:** Scena renderowana normalnie z tłem PDF/IMG
- **Bez podkładu:** Przezroczyste tło (tylko strefy i metki)

Implementacja polega na ustawieniu flagi `forceHideUnderlay` przed eksportem, co powoduje, że warstwa podkładu (`KonvaImage`) ma `visible={false}`.

## 3. Mechanizm Eksportu DXF

### 3.1 Różnica w Porównaniu do PNG

W przeciwieństwie do PNG (który jest zrzutem rastrowym), eksport DXF wymaga:
1. **Filtrowania geometrii:** Tylko obiekty wewnątrz kadru powinny być eksportowane
2. **Przekształcenia współrzędnych:** Punkt (0,0) pliku DXF powinien odpowiadać lewemu górnemu rogowi kadru

### 3.2 Sygnatura Funkcji

```typescript
// src/lib/dxfExport.ts
export function exportToDXF(
  floorState: FloorCanvasState,
  zones: Record<string, ZoneData>,
  getTagText: (zoneId: string) => { col1: string; col2: string },
  exportFrame?: { x: number; y: number; width: number; height: number }
): string
```

### 3.3 Walidacja i Filtrowanie Geometrii

#### 3.3.1 Funkcje Walidacyjne

```typescript
// Walidacja liczby - chroni przed NaN/Infinity
function isValidNumber(val: number): val is number {
  return typeof val === 'number' && !isNaN(val) && isFinite(val);
}

// Bezpieczne zaokrąglanie
function safeToFixed(val: number, decimals: number = 2): number {
  if (!isValidNumber(val)) return 0;
  const mult = Math.pow(10, decimals);
  return Math.round(val * mult) / mult;
}

// Test AABB dla poligonów
function isPolygonInFrame(
  points: number[],
  frame: ExportFrame
): boolean {
  for (let i = 0; i < points.length; i += 2) {
    const px = points[i];
    const py = points[i + 1];
    if (
      isValidNumber(px) && isValidNumber(py) &&
      px >= frame.x && px <= frame.x + frame.width &&
      py >= frame.y && py <= frame.y + frame.height
    ) {
      return true;
    }
  }
  return false;
}
```

#### 3.3.2 Bezpieczne Przetwarzanie Punktów

Wszystkie punkty są filtrowane przed dodaniem do DXF:

```typescript
const cadPoints: { x: number, y: number }[] = [];
for (let i = 0; i < poly.points.length; i += 2) {
  const px = poly.points[i];
  const py = poly.points[i + 1];
  if (isValidNumber(px) && isValidNumber(py)) {
    cadPoints.push(toCAD({ x: px, y: py }));
  }
}

// Pomijamy poligony z mniej niż 3 punktami
if (cadPoints.length < 3) return;

// Dodatkowe filtrowanie - tylko prawidłowe punkty CAD
const validPoints = cadPoints.filter(p => isValidNumber(p.x) && isValidNumber(p.y));
if (validPoints.length < 3) return;
```

#### 3.3.3 Obsługa Błędów

Operacje które mogą się nie powieść (Hatch, MText) są opakowane w try-catch:

```typescript
try {
  dxf.addHatch(paths, { name: HatchPredefinedPatterns.SOLID }, {
    layerName: "WENTCAD_WYPELNIENIA",
    colorNumber: 7
  });
} catch (e) {
  // Hatch może się nie udać dla niektórych poligonów - pomijamy cicho
  console.warn('Hatch failed for zone:', zone.id);
}
```

### 3.4 Warstwy DXF

Definiowane NA POCZĄTKU funkcji (przed jakimkolwiek rysowaniem):

```typescript
dxf.addLayer("WENTCAD_PUNKT_ZERO", Colors.White);
dxf.addLayer("WENTCAD_OBRYSY", Colors.Blue);
dxf.addLayer("WENTCAD_WYPELNIENIA", Colors.Cyan);
dxf.addLayer("WENTCAD_METKI_RAMKI", Colors.Yellow);
dxf.addLayer("WENTCAD_METKI_TEKST", Colors.White);
dxf.addLayer("WENTCAD_KADRY", Colors.Green);  // Ramka kadrowania
```

### 3.5 Ograniczenia Cięcia Geometrii Wektorowej

**Ważne zastrzeżenia:**

1. **Test AABB jest przybliżony:** Poligon może mieć część wierzchołków wewnątrz kadru, a część na zewnątrz. W takim przypadku eksportujemy CAŁY poligon (nie obcinamy go geometrycznie). Użytkownik musi sam zdecydować, czy chce eksportować pełny obiekt.

2. **Metki poza kadrem:** Jeśli pozycja metki (`tagPosition` w `ZoneData`) znajduje się poza kadrem, metka NIE zostanie wyeksportowana (nawet jeśli poligon jest wewnątrz kadru).

3. **Podkład:** DXF NIE zawiera podkładu (PDF/IMG). Jest to celowe zachowanie - plik wektorowy zawiera tylko warstwy WENTCAD (obrysy, wypełnienia, metki).

## 4. Struktura Komponentów (Workspace2D)

### 4.1 Refs do Warstw

```typescript
// Workspace2D.tsx
const stageRef = useRef<Konva.Stage>(null);
const uiLayerRef = useRef<Konva.Layer>(null);      // Siatka, ramki, znaczniki
const contentLayerRef = useRef<Konva.Layer>(null);  // Strefy, metki
```

### 4.2 Struktura Renderowania

```tsx
<Stage ref={stageRef} ...>
  {/* Warstwa UI - ukrywana przed eksportem */}
  <Layer ref={uiLayerRef} name="ui">
    {drawGrid()}
    {/* Podkład */}
    {/* Znaczniki punktu 0 */}
    {/* Ramki kadru */}
    {/* Linie kalibracji */}
    {/* Pomiary */}
  </Layer>
  
  {/* Warstwa Content - zawsze eksportowana */}
  <Layer ref={contentLayerRef} name="content">
    {/* Poligony stref */}
    {/* Obrysy CAD */}
    {/* Metki */}
  </Layer>
</Stage>
```

### 4.3 Handlery Eksportu

#### handleExportPNG
Pełna implementacja znajduje się w sekcji 2.2.

#### handleEditRegion
Odpowiada za edycję zapisanego kadru:

```typescript
const handleEditRegion = useCallback((region) => {
  setIsExportModalOpen(false);
  if (activeFloorId) {
    setCurrentTool(activeFloorId, 'CROP');
    setCurrentPolygonPoints([{ x: region.x, y: region.y }]);
    setIsDrawingPolygon(true);
  }
}, [activeFloorId]);
```

#### handleDeleteRegion
Usuwa kadr z listy:

```typescript
const handleDeleteRegion = useCallback((regionId: string) => {
  const filtered = activeFloorMetadata.exportRegions.filter(r => r.id !== regionId);
  updateFloor(activeFloorId, { exportRegions: filtered });
}, [activeFloorId, activeFloorMetadata, updateFloor]);
```

## 6. Weryfikacja i Testy

### 6.1 Test PNG Export
1. Narysuj kadr na scenie z widocznymi strefami
2. Otwórz okno eksportu (przycisk "📷 Eksport" w toolbarze tabeli)
3. Wybierz kadr z listy i kliknij "Eksportuj do PNG"
4. Sprawdź, czy:
   - Siatka NIE jest widoczna na eksporcie
   - Ramki kadru NIE są widoczne na eksporcie
   - Obraz odpowiada dokładnie zaznaczonemu obszarowi
   - Strefy są wyraźne (opcja z/bez podkładu)
   - Rozdzielczość jest wysoka (pixelRatio: 3)

### 6.2 Test DXF Export
1. Zdefiniuj kadr obejmujący część stref
2. Wyeksportuj do DXF
3. Otwórz plik w AutoCAD/QCAD/FreeCAD
4. Sprawdź, czy:
   - Plik otwiera się bez błędów
   - Tylko strefy wewnątrz kadru są widoczne
   - Współrzędne są poprawne (nie NaN)
   - Metki są poprawnie pozycjonowane
   - Ramka kadru jest widoczna jako zielona linia

### 6.3 Test Edycji Kadrów
1. Zapisz kilka kadrów
2. Otwórz okno eksportu
3. Kliknij "Edytuj" przy wybranym kadrze
4. Sprawdź, czy:
   - Modal się zamyka
   - Narzędzie CROP jest aktywne
   - Stary kadr jest widoczny do edycji
