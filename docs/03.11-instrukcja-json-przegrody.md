# Instrukcja Tworzenia Pliku JSON z Przegrodami Budowlanymi (WATT)

Dokument opisuje strukturę danych niezbędną do zdefiniowania biblioteki przegród budowlanych (ścian, stropów, dachów) w module WATT projektu WENTCAD.

## 1. Struktura Danych

Plik powinien zawierać listę obiektów lub oddzielne sekcje dla materiałów, zestawów warstw i typów przegród. W obecnej implementacji system korzysta z:
- `materials.json` – baza surowych materiałów.
- Stanu projektu (Zustand store) – gdzie przechowywane są konkretne definicje przegród.

### A. Materiał (`IfcMaterial`)
Definiuje właściwości fizyczne pojedynczego materiału.

| Pole | Typ | Opis | Jednostka |
| :--- | :--- | :--- | :--- |
| `id` | `string` | Unikalny identyfikator (np. UUID lub klucz tekstowy) | - |
| `name` | `string` | Nazwa wyświetlana materiału | - |
| `thermalConductivity` | `number` | Współczynnik przewodzenia ciepła ($\lambda$) | W/(m·K) |
| `massDensity` | `number` | Gęstość objętościowa ($\rho$) | kg/m³ |
| `specificHeatCapacity`| `number` | Ciepło właściwe ($c_p$) | J/(kg·K) |

**Przykład:**
```json
{
  "id": "mat-brick-solid",
  "name": "Cegła ceramiczna pełna",
  "thermalConductivity": 0.77,
  "massDensity": 1800,
  "specificHeatCapacity": 880
}
```

### B. Warstwa Materiału (`IfcMaterialLayer`)
Pojedynczy komponent wewnątrz przegrody.

| Pole | Typ | Opis | Jednostka |
| :--- | :--- | :--- | :--- |
| `id` | `string` | Unikalny identyfikator warstwy | - |
| `materialId` | `string` | Referencja do `id` materiału z bazy | - |
| `thickness` | `number` | Grubość warstwy | m |

### C. Zestaw Warstw (`IfcMaterialLayerSet`)
Logicze zgrupowanie warstw tworzących strukturę przegrody.

| Pole | Typ | Opis |
| :--- | :--- | :--- |
| `id` | `string` | Unikalny identyfikator zestawu |
| `name` | `string` | Nazwa zestawu warstw |
| `layers` | `array` | Tablica obiektów `IfcMaterialLayer` (kolejność od wewnątrz) |

### D. Typ Przegrody (`IfcWallType`)
Finalna definicja przegrody przypisywana do krawędzi strefy.

| Pole | Typ | Opis |
| :--- | :--- | :--- |
| `id` | `string` | Unikalny identyfikator typu |
| `name` | `string` | Nazwa przegrody (np. "Ściana Zewnętrzna SZ1") |
| `layerSetId` | `string` | Referencja do `id` zestawu warstw (`IfcMaterialLayerSet`) |
| `predefinedType` | `string` | Typ: `STANDARD`, `SOLIDWALL`, `PARTITIONING` |
| `isExternal` | `boolean` | Czy przegroda styka się z powietrzem zewnętrznym |

## 2. Kompleksowy Przykład (JSON)

Poniżej znajduje się przykład struktury, którą można zaimportować do systemu jako zestaw gotowych przegród:

```json
{
  "wallTypes": [
    {
      "id": "wall-ext-01",
      "name": "Ściana zewnętrzna 24+15",
      "layerSetId": "ls-ext-01",
      "predefinedType": "STANDARD",
      "isExternal": true
    }
  ],
  "layerSets": [
    {
      "id": "ls-ext-01",
      "name": "Warstwy Ściany Zewnętrznej",
      "layers": [
        { "id": "l1", "materialId": "mat-plaster-gypsum", "thickness": 0.015 },
        { "id": "l2", "materialId": "mat-brick-hollow", "thickness": 0.24 },
        { "id": "l3", "materialId": "mat-eps-std", "thickness": 0.15 },
        { "id": "l4", "materialId": "mat-plaster-cement", "thickness": 0.01 }
      ]
    }
  ]
}
```

## 3. Ważne Uwagi
1. **Kolejność Warstw**: Warstwy w tablicy `layers` powinny być podawane **od strony wewnętrznej do zewnętrznej**. Jest to kluczowe dla poprawnego obliczania współczynnika $U$ oraz analizy cieplnej.
2. **Jednostki**: Grubość podajemy w **metrach** (np. `0.24` dla 24 cm). Współczynniki fizyczne ($\lambda$, $\rho$, $c_p$) muszą być zgodne z układem SI.
3. **ID**: Każdy element musi posiadać unikalne `id`. W przypadku tworzenia plików ręcznie, można używać czytelnych kluczy tekstowych, o ile są unikalne w skali całego projektu.
