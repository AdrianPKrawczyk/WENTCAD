# 00.1-project-management-versioning.md - Zarządzanie Projektami i Wehikuł Czasu

> **[DIRECTIVE FOR AGENT]** Pominąłeś infrastrukturę zarządzania projektami. Obecnie aplikacja działa w trybie "Single Project", co uniemożliwia profesjonalną pracę. Musisz wdrożyć warstwę nadrzędną (Project Dashboard) oraz system migawek (Snapshots).

## 1. STRUKTURA BAZY (SUPABASE) - WERYFIKACJA
Upewnij się, że tabele są zgodne z `migration_v0.sql`:
- `projects`: Główny rekord projektu.
- `project_versions`: Tabela na "Snapshopy" (Snapshoty) stanu `state_data`.

## 2. MODUŁ: PROJECT DASHBOARD (EKRAN STARTOWY)
1. **Overlay/Modal Startowy:** Przy uruchomieniu aplikacji (jeśli nie wybrano projektu), wyświetl modal z listą projektów użytkownika.
2. **Akcje:**
   - "Nowy Projekt": Inicjalizuje pusty stan i tworzy rekord w `projects`.
   - "Otwórz": Pobiera `state_data` i zasila nim wszystkie Store'y (Zone, Floor, Duct).

## 3. MODUŁ: WEHIKUŁ CZASU (VERSIONING UI)
1. **Panel Wersji (RightDrawer lub TopBar):** Sekcja "Historia Wersji".
2. **Tworzenie Wersji:** Przycisk "Zapisz Snapshot" (użytkownik podaje nazwę, np. "Po spotkaniu z architektem"). Program kopiuje obecny stan `state_data` do tabeli `project_versions`.
3. **Przywracanie:** Kliknięcie w starą wersję musi nadpisać obecny stan aplikacji (z ostrzeżeniem "Utracisz niezapisane zmiany").

## 4. AUTO-ZAPIS (SILENT SYNC)
1. **Debounce:** Każda zmiana w dowolnym Store (Zone/Floor) wyzwala `update` rekordu w tabeli `projects` w kolumnie `state_data` po 3 sekundach bezczynności.
2. **Indykator:** W StatusBar wyświetlaj ikonę: "Chmura (Sync...)" -> "Chmura (Zapisano)".

## 5. INTEGRACJA Z UI
- W TopBarze dodaj nazwę aktualnego projektu. Kliknięcie w nią pozwala wrócić do Dashboardu (Listy projektów).
